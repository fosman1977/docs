<Note>
Import client, contact, and historical data from CSV files into the Chambers Platform.
</Note>

## Overview

The import system supports:

- **Lex Exports** - Automatic field mapping for Lex practice management data
- **Generic CSV** - Any CSV with manual field mapping
- **Client Data** - Company and contact information
- **Lead History** - Historical matter data

## Accessing Import

Navigate to **Clerk** → **Import** or **Admin** → **Data Import**

## Supported Import Types

| Type | Description | Auto-mapping |
|------|-------------|--------------|
| Lex Contacts | Contact export from Lex | Yes |
| Lex Matters | Matter history from Lex | Yes |
| Generic Clients | Any client CSV | Manual |
| Generic Contacts | Any contact CSV | Manual |

## Preparing Your Data

### From Lex

<Steps>
  <Step title="Open Lex">
    Launch your Lex practice management system
  </Step>
  <Step title="Navigate to Export">
    Go to the section you want to export (Contacts, Matters, etc.)
  </Step>
  <Step title="Select Records">
    Choose which records to include
  </Step>
  <Step title="Export as CSV">
    Use the export function and save as CSV format
  </Step>
  <Step title="Save File">
    Save to your computer for upload
  </Step>
</Steps>

### From Other Systems

Ensure your CSV includes:

**Required columns:**
- Name or Company Name
- At least one identifier (email, phone, or reference)

**Recommended columns:**
- Email address
- Phone number
- Address
- Category/Type
- Notes

<Tip>
Use the first row for column headers. The system will use these to suggest field mappings.
</Tip>

## Import Process

<Steps>
  <Step title="Upload File">
    Click "Upload CSV" and select your file
  </Step>
  <Step title="Select Import Type">
    Choose the type of data you're importing
  </Step>
  <Step title="Map Fields">
    Match CSV columns to platform fields. Lex exports auto-map; others require manual mapping.
  </Step>
  <Step title="Configure Options">
    Set duplicate handling and other options
  </Step>
  <Step title="Preview">
    Review the first 10 records before importing
  </Step>
  <Step title="Import">
    Click "Import" to process all records
  </Step>
</Steps>

## Field Mapping

### Lex Auto-Mapping

| Lex Field | Platform Field |
|-----------|----------------|
| Company Name | Client Name |
| Contact Name | Contact Full Name |
| Email | Primary Email |
| Phone | Primary Phone |
| Address | Address |
| Category | Client Category |
| Notes | Notes |
| Matter Ref | Lead Reference |
| Fee Earner | Assigned Barrister |

### Manual Mapping

For non-Lex CSVs:

1. Each CSV column is shown
2. Select the matching platform field from dropdown
3. Use "Skip" for columns you don't want to import
4. Required fields must be mapped

## Duplicate Handling

Choose how to handle records that match existing data:

| Option | Behaviour |
|--------|-----------|
| Skip Duplicates | Ignore matching records |
| Update Existing | Merge new data into existing records |
| Create New | Always create new records (may cause duplicates) |
| Flag for Review | Import but mark for manual review |

### Duplicate Detection

Records are matched by:
1. Email address (primary match)
2. Company name + contact name
3. Phone number
4. Reference number (if provided)

## Import Results

After import completes:

### Summary

Download a report of failed records with:

<Warning>

</Warning>

After importing:

<Steps>

  </Step>
  <Step title="Verify Mappings">
    Check client-barrister relationships are correct
  </Step>
</Steps>

## Troubleshooting

### Common Issues

**File won't upload**
- Check file is CSV format (not Excel .xlsx)
- Ensure file size is under 10MB
- Try saving with UTF-8 encoding

**Fields not mapping correctly**
- Check for extra spaces in headers
- Ensure consistent date formats
- Verify no special characters in headers

**Too many duplicates**
- Review duplicate detection settings
- Consider cleaning data before import
- Use "Flag for Review" option

**Import errors**
- Check required fields are populated
- Verify email formats are valid
- Ensure dates are in recognised format

### Large Imports

For imports over 1,000 records:

- Import in batches of 500-1,000
- Allow time between batches
- Monitor system performance

<Info>
Large imports run in the background. You'll receive a notification when complete.
</Info>
